# Pet Health Tracker Backend Migrations

Каталог содержит миграции базы данных backend-модуля.
Backend является владельцем схемы данных продукта.

## Назначение

1. Создавать таблицы продукта.
2. Изменять таблицы продукта.
3. Создавать индексы и ограничения.
4. Фиксировать изменения контракта хранения.
5. Подготавливать данные, необходимые для работы приложения.

## Правила

1. Изменение схемы базы данных должно оформляться миграцией.
2. Миграции должны быть применимы в локальном и production окружении.
3. Миграции не должны содержать реальные медицинские данные.
4. Имя миграции должно начинаться с временной метки.
5. Имя миграции должно описывать изменение контракта.

## Пример имени

```text
20260531000000_create_pet_profiles.sql
```
