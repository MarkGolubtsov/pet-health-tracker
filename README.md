# Cat Health Tracker

Cat Health Tracker - портал для учета и анализа состояния кота.
Проект хранит наблюдения владельца, лабораторные анализы, вес, препараты,
визиты к ветеринару и контекстные заметки.
Основной сценарий MVP - загрузка анализа, проверка распознанных показателей
и просмотр динамики значений во времени.

## Статус

Репозиторий содержит продуктовую спецификацию.
Архитектура, стек, интерфейсы и модель данных еще не зафиксированы.
README описывает внешний контракт продукта, а не внутреннюю реализацию.

## Цели MVP

1. Сохранить профиль кота и медицинский контекст.
2. Загрузить лабораторный бланк в поддерживаемом формате.
3. Извлечь показатели анализа в структурированном виде.
4. Дать владельцу возможность проверить и исправить результат распознавания.
5. Показать динамику показателей по датам анализов.
6. Отметить значения, которые вышли за референсный диапазон.
7. Экспортировать отчет для ветеринара.

## Профиль кота

Профиль содержит данные, которые нужны для интерпретации истории наблюдений.
Минимальный набор полей MVP:

1. Имя.
2. Дата рождения или возраст.
3. Порода.
4. Текущий вес.
5. Хронические болезни.
6. Текущие препараты.
7. Комментарии владельца.

## Загрузка анализов

Пользователь загружает лабораторный бланк и связывает его с профилем кота.
MVP должен поддерживать следующие типы входных документов:

1. Фото.
2. PDF.
3. Скан лабораторного бланка.

После загрузки документ получает статус обработки.
Ошибки распознавания должны быть видны пользователю до сохранения результата.

## Распознавание показателей

OCR должен извлекать показатели анализа в структурированном формате.
Для каждого показателя сохраняются следующие поля:

1. Название показателя.
2. Значение.
3. Единица измерения.
4. Референсный диапазон.
5. Дата анализа.
6. Лаборатория.
7. Исходный документ.

MVP должен поддерживать распространенные показатели:

1. `ALT`.
2. `AST`.
3. `CREA`.
4. `UREA`.
5. `WBC`.
6. Глюкоза.
7. Эритроциты.
8. Тромбоциты.

Система не должна скрывать неопознанные или сомнительные значения.
Такие значения должны попадать в интерфейс проверки.

## Проверка результата

Перед сохранением пользователь проверяет таблицу распознанных данных.
Интерфейс проверки должен позволять изменить ключевые поля показателя:

1. Название.
2. Значение.
3. Единицу измерения.
4. Референсный диапазон.
5. Дату анализа.

Сохранение результата фиксирует проверенную версию анализа.
Исходный документ должен оставаться связанным с сохраненным анализом.

## Динамика показателей

Портал строит график значения по каждому показателю во времени.
График должен использовать дату анализа, а не дату загрузки документа.
Для каждой точки графика должен быть доступен контекст владельца.

В MVP приоритет имеют следующие показатели:

1. Креатинин.
2. Мочевина.
3. `ALT`.
4. `AST`.
5. Глюкоза.
6. Лейкоциты.
7. Эритроциты.
8. Тромбоциты.

## Статусы значений

Сервис не ставит диагнозы.
Он показывает наблюдаемый статус показателя относительно референса и истории.
Поддерживаемые статусы MVP:

1. В норме.
2. Выше референса.
3. Ниже референса.
4. Значительно изменилось относительно прошлого анализа.

Формулировки должны быть нейтральными.
Допустимый пример:

```text
Креатинин вырос на 32% относительно прошлого анализа.
Рекомендуется обсудить результат с ветеринаром.
```

Недопустимые формулировки:

1. `У кота ХПН`.
2. `Печеночная недостаточность`.
3. `Нужно срочно лечить`.

## Комментарии владельца

Комментарий связывается с конкретной датой анализа или событием.
Он нужен для объяснения изменений в динамике.
Примеры допустимого контекста:

1. Прием антибиотиков.
2. Период после операции.
3. Смена корма.
4. Рвота.
5. Капельницы.

## Экспорт для ветеринара

Пользователь должен иметь возможность сформировать PDF-отчет.
Отчет должен включать данные, которые можно проверить без доступа к порталу:

1. Профиль кота.
2. Период отчета.
3. Графики выбранных показателей.
4. Таблицу значений.
5. Референсные диапазоны.
6. Комментарии владельца.
7. Даты анализов.

## Базовые сценарии

1. Создать профиль кота.
2. Загрузить лабораторный бланк.
3. Проверить распознанные показатели.
4. Исправить ошибки распознавания.
5. Сохранить анализ.
6. Просмотреть динамику по показателям.
7. Добавить комментарий к дате анализа.
8. Экспортировать отчет для ветеринара.
